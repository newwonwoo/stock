## 🎯 Claude Code에게 — 시작 안내

이 repo를 클론한 시점부터 다음 순서로 진행해주세요.

1. **`DESIGN.md` 정독** — 전체 설계와 모든 모듈의 상세 명세
2. **`TASKS.md` 확인** — 구현 작업 체크리스트 (의존성 순서대로 정렬됨)
3. **첫 작업부터 순서대로 구현** — 각 작업 완료 시 체크박스 갱신
4. **모든 작업 완료 후 자체 검증** — `DESIGN.md` 12장 품질확인서 작성

### 사용자(조정승) 작업 스타일

- **바이브 코딩**: 코드를 직접 안 만지고 자연어로 지시
- **질문 시 한국어**, 직설적·간결하게
- **어려운 용어는 풀어서** 설명
- **세션 종료 멘트 ❌** — 작업이 남아있으면 계속 진행
- **근본 해결 우선** — 에러 숨기기·우회 ❌, 원인 제거

### 핵심 원칙

- **추가 비용 0원** (사용자 의지). Claude API 호출 ❌, 외부 유료 서비스 ❌
- **Claude Max 플랜** 안에서 모든 분석 처리
- **분석은 GitHub Actions가 데이터 수집·정리만**, **Claude.ai 루틴이 분석·발송**
- 모든 모듈은 **단위 테스트 가능**한 형태로 작성

### 검증 프로토콜 (수정 후 매번 필수)

1. `python -c "import ast; ast.parse(open('파일.py').read())"` — 문법 체크
2. import 검증
3. 핵심 함수 mock 실행
4. 미정의 변수 grep
5. 변수 정의 확인

문법 체크만으로는 `NameError`를 못 잡습니다. 5단계 모두 필수.
